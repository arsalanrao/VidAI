VidAiPro — Complete Technical Specification

v1.0 Document date: June 19, 2026  |  Product: Automated viral YouTube Shorts from reference videos  |  Stack: React Native 0.86 + FastAPI + Celery + GPU workers

1. Project Overview

1.1 Vision

VidAiPro lets a creator paste a viral YouTube URL and automatically produce a new, original Short inspired by the reference video's structure — not a remix of the original footage.

Input: https://youtube.com/watch?v=...
Output: Finished vertical Short (9:16) with new title, hook, narration, visuals, thumbnail — ready for YouTube upload after user approval.

1.2 Core Principle: Rewrite, Don't Remix

DoDon't
Extract transcript, hook, pacing, structureReuse original video/audio
Rewrite script with Kimi K2.6Copy sentences verbatim
Generate new images and video clipsDownload and edit source video
Create new thumbnailSteal original thumbnail pixels

4. Pipeline Stages (Detailed)

Stage 1: YouTube Extraction

Tools: yt-dlp, ffmpeg, Whisper (fallback)
Output: source_manifest.json with transcript, captions, thumbnail, duration, metadata.

Stage 2: Kimi K2.6 Creative Package

Model: kimi-k2.6 @ https://api.moonshot.ai/v1
Config: thinking: { type: "disabled" } for fast JSON.

{
  "newTitle": "Scientists Fear These 5 Sea Monsters",
  "newHook": "Number 3 has killed more people than sharks.",
  "description": "...",
  "tags": ["sea monsters", "ocean", "shorts"],
  "narration": "Full voiceover script...",
  "thumbnailPrompt": "Dramatic underwater scene...",
  "scenes": [
    {
      "id": 1,
      "duration": 4,
      "narration": "You won't believe what lurks beneath the waves.",
      "imagePrompt": "Dark deep ocean, 9:16, no text",
      "needs_lip_sync": false,
      "style": "b-roll"
    }
  ]
}

Stage 3: Parallel Generation

  • 3a Thumbnail: FLUX.2 Klein 4B × 3 → thumb_0..2.jpg
  • 3b TTS: Magpie Multilingual → narration.wav
  • 3c Keyframes: FLUX per scene → scene_01.png...
After TTS, measure audio duration and proportionally adjust scene durations to match.

Stage 4: Scene Video

V1: FFmpeg Ken Burns zoom/pan on each keyframe.
V2: Cosmos-Predict2-2B-Video2World (image + prompt → clip).

Stage 5: Lip Sync (V2 only, BEFORE merge)

Only when needs_lip_sync: true. Use Wav2Lip or MuseTalk per talking-head scene — never on merged B-roll.

Stage 6: FFmpeg Final Merge

Concat scenes + mux narration + (V2) subs + BGM → final.mp4

Stage 7–8: Preview & Upload

User watches preview → approves → backend uploads via YouTube Data API v3 (resumable).

7. FFmpeg Workflow

7.1 Ken Burns (V1)

ffmpeg -y -loop 1 -i scene_01.png \
  -vf "scale=1216:2160:force_original_aspect_ratio=increase,crop=1080:1920,\
       zoompan=z='min(1+0.001*on,1.2)':d=120:x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)':s=1080x1920:fps=30" \
  -t 4 -an -c:v libx264 -preset fast -crf 23 -pix_fmt yuv420p scene_01.mp4

7.2 Concat + Audio Mux

ffmpeg -y -f concat -safe 0 -i concat_list.txt -c copy all_scenes.mp4
ffmpeg -y -i all_scenes.mp4 -i narration.wav -c:v copy -c:a aac -b:a 192k -shortest final.mp4

7.3 YouTube Normalize

ffmpeg -y -i final.mp4 -c:v libx264 -preset slow -crf 18 \
  -c:a aac -b:a 192k -ar 48000 -movflags +faststart -pix_fmt yuv420p final_youtube.mp4

8. YouTube Upload Flow

  1. Create GCP project → enable YouTube Data API v3
  2. OAuth 2.0 credentials (backend only — never in mobile app)
  3. Scopes: youtube.upload, youtube
  4. Store encrypted refresh_token in PostgreSQL
  5. Resumable upload: videos.insert with 10 MB chunks
  6. Separate call: thumbnails.set
  7. Default privacy: private

11. Database Design

Core tables: users, youtube_tokens, jobs, job_scenes, job_assets, job_events.

Jobs store source_manifest (JSONB), creative_package (JSONB), publish metadata, YouTube video ID. Scenes store per-scene prompts, durations, asset paths, lip sync flag.
